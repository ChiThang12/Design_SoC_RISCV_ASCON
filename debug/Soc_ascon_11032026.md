# Debug Log — RISC-V SoC + ASCON IP Firmware

**Project:** RISC-V SoC v3.0 — AXI4 Crossbar 3Mx5S + ASCON-128 DMA mode  
**Firmware:** `main.c` — ASCON encrypt via DMA engine  
**Toolchain:** `riscv64-unknown-elf-gcc 13.2.0` — `-march=rv32im -mabi=ilp32 -O0`

## Lịch sử lỗi và cách sửa

---

### Lỗi #1 — PTEXT ghi sai địa chỉ `0x48657000`

**Triệu chứng:**
```
[ST] addr=0x48657000  data=0x48656c6c   ← "Hell" ghi vào địa chỉ rác
[ST] addr=0x48657004  data=0x6f210000
[!!!] DECERR count: 7
DMEM[0x10000000] = 0x00000000           ← plaintext trống
```

**Nguyên nhân:**  
`static const uint32_t PTEXT[4]` khiến GCC đặt data vào section `.srodata` (small read-only data). Linker script chỉ có `*(.rodata*)`, không capture `.srodata` → linker đặt section sai vị trí. GCC tính địa chỉ PTEXT bằng `lui+addi` dựa trên vị trí `.srodata` trong binary → offset sai → CPU ghi vào `0x48657000` (chính là giá trị `0x4865` = "He" của PTEXT bị nhầm thành địa chỉ).

**Fix:**  
Đổi `static const uint32_t[]` sang `#define`:

```c
// TRƯỚC — GCC sinh .srodata, địa chỉ bị tính sai
static const uint32_t PTEXT[2] = { 0x48656C6Cu, 0x6F210000u };

// SAU — GCC embed thẳng vào li/addi, không cần load từ memory
#define PTEXT_0  0x48656C6Cu
#define PTEXT_1  0x6F210000u
```

Assembly kết quả:
```asm
li  a4, 0x48656C6C    ← giá trị embed trực tiếp vào instruction
sw  a4, 0(a5)         ← ghi vào DMEM đúng địa chỉ
```

**Cũng sửa thêm trong `compile_c_to_hex.sh`:**
- Linker script: thêm `*(.srodata*)` và `*(.sdata2*)` vào ROM section
- objcopy: đổi từ `--only-section` sang `--remove-section` để tự động capture mọi section nhỏ
- Compile flag: thêm `-mno-relax -fno-pic`

**Fix:** Cùng fix với Lỗi #1 — đổi toàn bộ sang `#define`.

### Lỗi #3 — `core_start=0`, `dma_start=0` — ASCON không phản hồi

**Triệu chứng:**
```
core_start pulses : 0
dma_start  pulses : 0
S2 ASCON accesses : 4   ← CPU ghi được vào ASCON nhưng không trigger
STATUS at halt    : 0x00000000
```

**Nguyên nhân:**  
Ghi `CTRL = CTRL_DMA_EN | CTRL_START = 0x05` trong một lần duy nhất. RTL có thể xử lý `DMA_EN` (bit 2) và `START` (bit 0) ở 2 clock edge khác nhau — `START` được sample trước khi `reg_dma_en` được latch → DMA không start, core không start.

**Fix:** Tách thành 2 lần ghi:

```c
// TRƯỚC
ASCON_CTRL = CTRL_DMA_EN | CTRL_START;   // 0x05 — không trigger

// SAU
ASCON_CTRL = CTRL_DMA_EN;                // latch reg_dma_en = 1
__asm__ volatile ("nop");
__asm__ volatile ("nop");
ASCON_CTRL = CTRL_DMA_EN | CTRL_START;   // START với DMA_EN ổn định
```

### Lỗi #4 — `core_start=1` nhưng `dma_start=0`

**Triệu chứng:**
```
core_start pulses : 1   ← core chạy (CPU-direct mode)
dma_start  pulses : 0   ← DMA không chạy
[ASCON] CORE DONE #1
DMEM[0x10000010] = 0x00000000   ← ciphertext trống
```

Store Scoreboard:
```
[0x20000000] = 0x00000001   ← CTRL chỉ có START bit, không có DMA_EN!
```

**Nguyên nhân:**  
Sau fix Lỗi #3, lần ghi thứ 2 chỉ là `CTRL = CTRL_START = 0x01`. RTL thấy `DMA_EN=0` trong write data → chỉ trigger **core** (CPU-direct mode), không trigger DMA engine.

**Fix:**

```c
// TRƯỚC (sai) — lần 2 thiếu DMA_EN
ASCON_CTRL = CTRL_START;             // 0x01 → core start, DMA không start

// SAU (đúng) — giữ DMA_EN trong cả 2 lần ghi
ASCON_CTRL = CTRL_DMA_EN | CTRL_START;   // 0x05 → DMA start
```

### Lỗi #5 (hiện tại) — `dma_start=1` nhưng `src=0x00000000`

**Triệu chứng:**
```
[ASCON] DMA START #1  src=0x00000000  dst=0x00000000  len=8
[DMA64-AR] addr=0x00000000  (64-bit side)
[WARN] M2(DMA) AR outside DMEM range! addr=0x00000000
dma_busy = 1   ← stuck, không bao giờ done
dma_done = 0
```

Store Scoreboard xác nhận CPU đã ghi đúng:
```
[0x20000100] = 0x10000000   ← DMA_SRC đúng ✓
[0x20000104] = 0x10000010   ← DMA_DST đúng ✓
[0x20000108] = 0x00000018   ← DMA_LEN=24 đúng ✓
```

Nhưng `DMA config at halt: src=0x00000000` — RTL đọc `reg_dma_src = 0`.

**Nguyên nhân:**  
Khi `CTRL=START` được assert, `ascon_dma` latch `reg_dma_src/dst/len` ngay tại clock edge đó. Tuy nhiên các AXI write ghi `DMA_SRC/DST/LEN` vào slave register chưa được commit (AW+W+B handshake chưa hoàn tất) → DMA latch giá trị cũ `0x00000000`.

Timeline:
```
cycle 238  ST → 0x20000100 = 0x10000000  (DMA_SRC write issued)
cycle 268  ST → 0x20000000 = 0x00000005  (CTRL=START)
           ↑ AXI write chưa chắc đã hoàn tất!
```

**Fix:** Thêm AXI write barrier (read-back fence) sau khi ghi DMA config:

```c
static void step4_config_dma(void)
{
    volatile uint32_t fence;

    ASCON_DMA_SRC = DMEM_BASE + DMEM_PTEXT_OFF;
    ASCON_DMA_DST = DMEM_BASE + DMEM_CTEXT_OFF;
    ASCON_DMA_LEN = DMA_TOTAL_LEN;

    // AXI write barrier: đọc lại DMA_SRC để force drain AXI write pipeline
    // CPU phải chờ read data về → đảm bảo mọi write trước đó đã commit
    fence = ASCON_DMA_SRC;
    (void)fence;
}
```

**Trạng thái:** Đang chờ xác nhận kết quả sau fix này.
